---
title: Email & Password
description: Email and password authentication with RustAuth.
---

RustAuth ships built-in email/password authentication on the core `RustAuth` builder — no plugin required.

<Callout type="info">
  For username-based sign-in, enable the [username plugin](/docs/plugins/username) alongside email/password.
</Callout>

## Enable

```toml
[dependencies]
rustauth = { version = "0.1", features = ["plugins", "sqlx-sqlite"] }
```

```rust
use rustauth::prelude::*;

let auth = RustAuth::builder()
    .secret(std::env::var("RUSTAUTH_SECRET")?)
    .base_url(std::env::var("RUSTAUTH_BASE_URL")?)
    .email_password(
        EmailPasswordOptions::new()
            .enabled(true)
            .auto_sign_in(true)
            .require_email_verification(false),
    )
    .build()
    .await?;
```

| Option | Purpose |
| ------ | ------- |
| `enabled` | Turn email/password routes on |
| `auto_sign_in` | Create a session after sign-up |
| `require_email_verification` | Block sign-in until email is verified |
| `min_password_length` / `max_password_length` | Password policy |

Verification and reset emails use outbound hooks — never block the HTTP response on send. See [outbound delivery security](/docs/security-outbound-delivery) and [email concepts](/docs/concepts/email).

## HTTP endpoints

| Method | Path | Description |
| ------ | ---- | ----------- |
| POST | `/sign-up/email` | Register with name, email, password |
| POST | `/sign-in/email` | Sign in with email and password |
| POST | `/sign-out` | End session |
| GET | `/get-session` | Read current session (cookie required) |

Request bodies use JSON with camelCase field names (`rememberMe`, etc.). Example sign-up:

```bash
curl -i -X POST "$RUSTAUTH_BASE_URL/sign-up/email" \
  -H 'content-type: application/json' \
  -d '{"name":"Ada","email":"ada@example.com","password":"secure-pass-123","rememberMe":true}'
```

Server-side helpers live in `examples/backend-reference/src/client/flows.rs`. See [basic usage](/docs/basic-usage).

## Client integration

RustAuth does not ship a browser `authClient`. Your frontend or BFF calls the REST endpoints above and stores the session cookie RustAuth sets on success.

## Database

Email/password uses core `user` and `account` tables. Run `rustauth db migrate --yes` after enabling plugins that extend the schema.
