---
title: signup_disabled
description: OAuth sign-in attempted for a new user while sign-up is disabled.
---

The provider account does not match an existing user and sign-up is disabled for this provider or flow.

## Rust implementation

**Emitted by RustAuth:** yes

| Location | When |
| -------- | ---- |
| `crates/rustauth-core/src/auth/oauth/account_linking.rs:155` | `disable_sign_up` is true and no user match |
| `crates/rustauth-core/src/auth/oauth/errors.rs:19` | Code string `signup_disabled` |

Redirect shape: `{errorCallbackURL}?error=signup_disabled`.

## Handling in Rust

Parse `error=signup_disabled` from the OAuth redirect URL.

## How to fix

- Allow sign-up on the provider (`disable_sign_up` / `disable_implicit_sign_up` in provider options).
- Invite users through an admin flow instead of open OAuth registration.
- Have users register with email/password first, then link OAuth while authenticated.
