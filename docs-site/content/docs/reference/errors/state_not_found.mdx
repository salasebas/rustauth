## Rust implementation

<Callout type="warn">
  Not yet emitted by RustAuth (parity gap). Upstream may redirect with `?error=state_not_found` when a database-stored verification row is missing. RustAuth core social flows emit **`invalid_state`** instead (`crates/rustauth-core/src/api/routes/social/flow.rs:127`). See [state_mismatch](/docs/reference/errors/state_mismatch) for the Rust mapping.
</Callout>

## Handling in Rust

Handle `invalid_state` on redirect URLs for core flows. JSON [`ApiErrorResponse`](/docs/concepts/api#error-handling) applies to REST endpoints.

## How to fix

- Same guidance as state verification failures: shared storage, avoid callback replay, stable secrets and domains.
