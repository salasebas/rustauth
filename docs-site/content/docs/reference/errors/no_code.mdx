---
title: no_code
description: OAuth callback arrived without an authorization code.
---

The identity provider redirected back without a `code` query parameter (user cancelled, provider error page, or malformed callback).

## Rust implementation

**Emitted by RustAuth:** yes

| Location | When |
| -------- | ---- |
| `crates/rustauth-core/src/api/routes/social/flow.rs:134` | `code` query param missing on social callback |
| `crates/rustauth-plugins/src/oauth_proxy/hooks.rs:183` | OAuth Proxy callback missing `code` |
| `crates/rustauth-sso/src/routes/oidc.rs:109` | SSO OIDC callback missing `code` |

Redirect shape: `{errorCallbackURL}?error=no_code`.

## Handling in Rust

Inspect the redirect `Location` header or browser URL for `error=no_code`. JSON [`ApiErrorResponse`](/docs/concepts/api#error-handling) applies to REST endpoints, not OAuth redirects.

## How to fix

- Confirm the provider is configured to return an authorization code grant.
- Check whether the user denied consent or closed the provider window.
- Verify callback URL registration matches your RustAuth `base_url` and provider settings.
