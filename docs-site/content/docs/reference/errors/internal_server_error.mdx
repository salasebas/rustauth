## Rust implementation

<Callout type="warn">
  Not yet emitted by RustAuth as an OAuth redirect slug named `internal_server_error` (parity gap). Unexpected failures in Rust typically surface as:
</Callout>

- **Router errors** — `RustAuthError` returned from `handler_async` (check server logs).
- **JSON API errors** — HTTP 500 with [`ApiErrorResponse`](/docs/concepts/api#error-handling) where implemented.
- **Plugin password validation** — `PluginPasswordValidationRejection::internal_server_error` in plugins such as Have I Been Pwned (returns plugin JSON codes, not this redirect slug).

## Handling in Rust

```rust
match auth.handler_async(request).await {
    Ok(response) if !response.status().is_success() => {
        let error: rustauth::api::ApiErrorResponse = serde_json::from_slice(response.body())?;
        tracing::error!(code = %error.code, message = %error.message, "auth API error");
    }
    Err(error) => tracing::error!(%error, "auth router error"),
    Ok(_) => {}
}
```
