title: Errors
description: Error codes returned by RustAuth HTTP and OAuth redirect flows.
---

RustAuth surfaces errors in two shapes:

1. **JSON API errors** — `{ "code": "INVALID_EMAIL", "message": "…" }` via [`ApiErrorResponse`](/docs/concepts/api#error-handling) on REST routes.
2. **OAuth redirect errors** — HTTP redirect to your `errorCallbackURL` (or `{base_url}/error`) with `?error=<slug>` during social/OAuth callbacks.

Run `docs-site/scripts/list-rust-error-codes.sh` from the repo root for a full Rust inventory before releases.

## Core auth (`error_codes.rs`)

| Code                                    | Message (summary)                       | Emitted by RustAuth |
| --------------------------------------- | --------------------------------------- | ------------------- |
| `INVALID_EMAIL`                         | Invalid email                           | yes                 |
| `INVALID_PASSWORD`                      | Invalid password                        | yes                 |
| `INVALID_EMAIL_OR_PASSWORD`             | Invalid email or password               | yes                 |
| `INVALID_TOKEN`                         | Invalid token                           | yes                 |
| `INVALID_REQUEST_BODY`                  | Invalid request body                    | yes                 |
| `FIELD_NOT_ALLOWED`                     | Field not allowed                       | yes                 |
| `USER_ALREADY_EXISTS`                   | User already exists                     | yes                 |
| `USER_ALREADY_EXISTS_USE_ANOTHER_EMAIL` | User already exists (use another email) | yes                 |
| `EMAIL_NOT_VERIFIED`                    | Email not verified                      | yes                 |
| `SESSION_EXPIRED`                       | Session expired                         | yes                 |
| `SESSION_NOT_FRESH`                     | Session not fresh                       | yes                 |
| `CREDENTIAL_ACCOUNT_NOT_FOUND`          | Credential account not found            | yes                 |
| `NOT_FOUND`                             | Not found                               | yes                 |
| `UNAUTHORIZED`                          | Unauthorized                            | yes                 |

Additional router codes live in `rustauth-core/src/api/error.rs` (`INVALID_ORIGIN`, `INVALID_CALLBACK_URL`, `TOO_MANY_REQUESTS`, etc.).

## Per-plugin error codes

Plugins register additional stable codes via `PluginErrorCode`. See each plugin doc for its error table:

* [Two-Factor](/docs/plugins/2fa) — `INVALID_OTP`, backup codes, etc.
* [Username](/docs/plugins/username) — `INVALID_USERNAME`, length/validation codes
* [Organization](/docs/plugins/organization) — org/member/invitation failures
* [API Key](/docs/plugins/api-key), [Admin](/docs/plugins/admin), [Device Authorization](/docs/plugins/device-authorization), [Passkey](/docs/plugins/passkey), [SSO](/docs/plugins/sso), [Stripe](/docs/plugins/stripe), and others

Use `list-rust-error-codes.sh` to enumerate plugin registrations across `crates/`.

## Unknown errors

If the `error` query parameter does not match a known slug, your app should show a generic failure UI. See the fallback [unknown](/docs/reference/errors/unknown) page (not listed in sidebar navigation).
