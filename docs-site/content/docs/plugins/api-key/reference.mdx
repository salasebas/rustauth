Reference for the API key plugin types in `rustauth::plugins::api_key`. For setup and routes, see the [API key overview](/docs/plugins/api-key) and [advanced guide](/docs/plugins/api-key/advanced).

## ApiKeyOptions

```rust
pub struct ApiKeyOptions {
    pub configurations: Vec<ApiKeyConfiguration>,
    pub schema: ApiKeySchemaOptions,
}
```

Factory: `api_key(options: ApiKeyOptions) -> Result<AuthPlugin, RustAuthError>`

Builder:

```rust
ApiKeyOptions::builder()
    .configuration(/* ApiKeyConfiguration */)
    .schema(/* ApiKeySchemaOptions */)
    .build()?
```

## ApiKeyReference

Where the key is anchored:

| Variant | Description |
| --- | --- |
| `User` | Key belongs to the authenticated user (default) |
| `Organization` | Key belongs to an organization; requires organization plugin |

Organization keys include `reference_id` pointing at the organization row.

## ApiKeyConfiguration

Primary configuration struct. Important fields:

| Field | Type | Default / notes |
| --- | --- | --- |
| `config_id` | `Option<String>` | Required with multiple configs |
| `reference` | `ApiKeyReference` | `User` |
| `api_key_headers` | `Vec<String>` | `["x-api-key"]` |
| `disable_key_hashing` | `bool` | `false` (dev only) |
| `default_key_length` | `usize` | 64 |
| `default_prefix` | `Option<String>` | Optional key prefix |
| `require_name` | `bool` | Require name on create |
| `enable_metadata` | `bool` | Allow metadata JSON |
| `key_expiration` | `ApiKeyExpirationOptions` | TTL defaults and bounds |
| `rate_limit` | `ApiKeyRateLimitOptions` | Per-key rate limit |
| `enable_session_for_api_keys` | `bool` | Global session hook |
| `default_permissions` | `Option<Value>` | Default permission JSON |
| `storage` | `ApiKeyStorageMode` | `Database` or `SecondaryStorage` |
| `starting_characters` | `StartingCharactersConfig` | Display prefix storage |

## ApiKeyRateLimitOptions

```rust
pub struct ApiKeyRateLimitOptions {
    pub enabled: bool,
    pub time_window: Duration,
    pub max_requests: u64,
}
```

Defaults: `enabled: true`, `time_window: 1 day`, `max_requests: 10`.

Persisted columns on `api_keys`: `rate_limit_enabled`, `rate_limit_time_window`, `rate_limit_max`, `request_count`, `remaining`, `last_request`.

## ApiKeyExpirationOptions

```rust
pub struct ApiKeyExpirationOptions {
    pub default_expires_in: Option<Duration>,
    pub disable_custom_expires_time: bool,
    pub min_expires_in_days: u32,
    pub max_expires_in_days: u32,
}
```

## StartingCharactersConfig

```rust
pub struct StartingCharactersConfig {
    pub should_store: bool,
    pub characters_length: usize,
}
```

When `should_store` is true, the first `characters_length` characters of the raw key are saved for UI display.

## ApiKeyStorageMode

| Variant | Description |
| --- | --- |
| `Database` | Primary adapter only |
| `SecondaryStorage` | Secondary store with optional DB fallback |

Related fields: `custom_storage`, `fallback_to_database`, `revalidate_secondary_against_database`, `defer_updates`.

## Schema

Default table name: **`api_keys`**

| Column | Description |
| --- | --- |
| `id` | Key ID |
| `config_id` | Configuration profile |
| `name` | Human-readable name |
| `start` | Starting characters for display |
| `prefix` | Key prefix |
| `key` | Hashed secret |
| `reference_id` | User or organization ID |
| `enabled` | Whether the key is active |
| `expires_at` | Expiration timestamp |
| `metadata` | Optional JSON metadata |
| `permissions` | Optional permission JSON |
| Rate limit / refill columns | Counters and windows |

Override table and column names with `ApiKeySchemaOptions`.

Run migrations after enabling:

```bash
rustauth db migrate
```
