title: Plugins
description: Official RustAuth server plugins for authentication, organizations, tokens, and enterprise features.
---

RustAuth extends core auth with **server plugins** registered on `RustAuth::builder()` via `.plugin(...)`. Plugins add HTTP routes, database tables, hooks, and rate-limit rules. There is no browser client SDK — frontends call the same REST API under your auth base path.

Enable the `plugins` feature on the `rustauth` crate, plus satellite features for enterprise plugins (`passkey`, `sso`, `scim`, `oauth-provider`, `stripe`, `i18n`).

```toml
[dependencies]
rustauth = { version = "0.1", features = ["plugins", "sqlx-sqlite"] }
```

See [concepts/plugins](/docs/concepts/plugins) and `examples/backend-reference/src/auth/plugins.rs` for a full wiring reference.

## Migrations

Plugins that add tables require schema updates:

```bash
rustauth db migrate --yes
```

See the full [database schema reference](/docs/reference/database-schema) for all tables and columns.

Configure `rustauth.toml` `[plugins].enabled` to match the plugins you register in code. See [database migrations](/docs/database-migrations).
