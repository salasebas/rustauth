---
title: Have I Been Pwned
description: Compromised-password checks via Have I Been Pwned for RustAuth server plugin.
---

The have-i-been-pwned plugin rejects passwords that appear in known breach corpora. It runs as middleware on configured password-setting routes and is disabled by default.

## Install

```toml
[dependencies]
rustauth = { version = "0.1", features = ["plugins"] }
```

## Enable

```rust
use rustauth::RustAuth;
use rustauth::plugins::prelude::*;

let auth = RustAuth::builder()
    .secret("secret-a-at-least-32-chars-long!!")
    .plugin(have_i_been_pwned(HaveIBeenPwnedOptions {
        enabled: true,
        ..HaveIBeenPwnedOptions::default()
    }))
    .build()
    .await?;
```

Set `enabled: true` to activate checks. By default the plugin is wired but inactive.

## Configuration

| Field                                 | Description                                                                         |
| ------------------------------------- | ----------------------------------------------------------------------------------- |
| `enabled`                             | Turn breach checks on or off (default `false`)                                      |
| `paths`                               | Routes to inspect (default `/sign-up/email`, `/change-password`, `/reset-password`) |
| `custom_password_compromised_message` | Override the error message shown to users                                           |
| `checker`                             | Custom `HaveIBeenPwnedChecker` instead of the built-in HIBP client                  |

Builder helpers: `.enabled(...)`, `.path(...)`, `.paths(...)`, `.checker(...)`.

## HTTP endpoints

This plugin does not register routes. It intercepts password submissions on configured paths before handlers run. Compromised passwords return error code `PASSWORD_COMPROMISED`.

## Database

No schema changes.
